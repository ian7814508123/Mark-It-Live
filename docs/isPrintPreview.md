### 列印預覽行為與設計規範

1. 功能描述
   - 點擊預覽按鈕後，進入列印預覽模式 (Print Preview Mode)。
   - 預覽模式下，透過 CSS 與 JS 模擬實體紙張 (A4/A3/Letter) 的邊界、方向與分頁效果。

2. 用途
   - 提升 UX：讓使用者可以「所見即所得」地預覽列印或 PDF 輸出的最終結果。
   - 避免浪費：避免列印出破版或不符合預期的文件，減少反覆測試列印的次數。
   - 支援標註：提供一個絕對座標鎖定的畫布，讓使用者在列印前繪製便利貼與註記。

3. 與深淺模式的配合 (深淺模式 x 預覽行為)
   - 系統狀態分成四種情況：(Light, Preview)、(Light, Not Preview)、(Dark, Not Preview)、(Dark, Preview)。

#### ⚠️ 絕對核心準則：有無開關列印預覽，都不該影響最終列印行為
   - **(L,1)(L,0)(D,0)(D,1) 四種情況下印出來的文件，都必須是相同的 Formal Document (白紙黑字)。**
   - **實體一致性**：無論使用者是在編輯模式直接按 `Ctrl+P`，還是在預覽模式按下匯出，系統都會在輸出的瞬間強制注入 `@page { margin: 0; }`，並依靠我們自訂的 CSS `padding` 來控制邊距。這是為了確保所有元素的「絕對座標」不會因為瀏覽器的自動縮放而漂移。

4. 代碼區塊斷行策略配合
   - 正常編輯模式：長文本程式碼由語言種類決定是否換行，並提供按鈕自由切換。
   - 預覽與列印模式 (Print / Preview)：強制開啟換行 (Wrap) 並且不顯示切換按鈕，避免長程式碼與 Log 被裁切。

5. 未來擴充遵循規範 (Future Guidelines)
   - **單一事實來源 (Single Source of Truth)**：任何對「列印輸出」排版的修改，必須同時考慮「列印預覽」中的呈現。預覽區塊的 DOM 結構與類別（如 `.print-paper`）必須在實際列印 (`@media print`) 時具備相同的效果。
   - **禁止依賴瀏覽器邊界**：未來的版面擴充（例如頁首、頁尾、浮水印）必須實作在 `.print-paper` 內部。絕對不能依賴瀏覽器的預設邊距。
   - **互動元件遮罩**：任何在預覽模式下可見的 UI 互動元件（如標註工具列、編輯按鈕），都必須加上 `print:hidden`，確保它們不會出現在最終輸出的 PDF 中。

## 🍃 主題 E：竹林禪意 (bamboozen) 【STATUS: PLANNED】

### 1. 設計理念與適用場景
* **核心意象**：石燈籠、白砂、苔蘚綠、竹編墨綠、原木色、清靜悠遠。
* **適用情境**：心靈日記、長文創作、讀書筆記、思考整理。
* **UX 目標**：採用極低對比度與冷沉靜色彩，模擬日式榻榻米與竹林小徑的幽靜氛圍，讓讀者心靈沉靜，提供極佳的護眼效果。

# 主題選擇器 UX 優化與未來功能發想

內建的主題（Theme）越來越多時，目前彈出視窗中「兩欄平鋪按鈕」的設計會面臨以下痛點：
- 1. 畫面擁擠 
- 2. 缺乏分類 
- 3. 載入與擴充成本

---
## 1. 介面設計（UI/UX）優化
### A. 分類標籤頁（Tabbed Categories）> 目前已實作完成
將主題依照「風格」或「使用場景」進行分類，利用左側側邊欄或頂部 Tab 切換，避免一次呈現太多選項。

分類範例：全部、簡約（Minimal）、專業（Academic/Dev）、藝術/復古（Creative）。

好處：使用者能快速縮小尋找範圍，且單一頁面只需渲染少數主題。

~~### B. 網格卡片-> 這應該也不太理想, 因為佔據空間~~ (已棄用)
放棄目前單純的文字按鈕，改用網格卡片（Grid View）設計。

每個卡片內包含：主題名稱、風格標籤，以及一個微縮版的效果縮圖（Thumbnail）。

當滑鼠懸停（Hover）或點擊卡片時，右側或背景可以直接預覽該主題套用在 Markdown 上的局部效果，減少使用者反覆點擊切換的成本。

### C.  下拉選單

## 2. 未來功能擴充（Advanced Features）
A. 收藏/常用主題（Favorites / Recently Used）當主題多達 20 個以上時，多數使用者常用的其實固定就那 2-3 個。可以在主題卡片右上角加上「星星（$\star$）」功能，讓使用者收藏常用主題。在最上方獨立一區「常用主題」，讓使用者無需每次都重新尋找。

B. 自訂主題 (Custom CSS / Theme Variable Editor) 【STATUS: PLANNED】
開放自訂主題以徹底解放自訂權，滿足極客與重度 Markdown 用戶：
1. **介面設計**：在主題選擇器（`ThemeGridSelector`）中新增「自訂」分類頁籤。當選取時，顯示「自訂設定面板」。
2. **雙軌制編輯模式**：
   * **🎨 視覺化引導（簡易模式）**：提供 Color Pickers 讓用戶直接點選修改核心變數（如背景色、文字顏色、強調色、程式碼背景、邊框色等）。
   * **💻 CSS 模板編輯（進階模式）**：提供 Textarea (預留 CodeMirror CSS 整合空間)，預先載入 CSS 變數模板 (如 `.theme-custom { ... }`)，讓使用者直接貼上自訂的 CSS 規則。
3. **技術實作**：
   * 將自訂配置 JSON 或純 CSS 儲存於 LocalStorage。
   * 套用主題時，動態將用戶自訂的 CSS 注入到 `<style id="markdown-custom-theme-{x}">` 中進行覆蓋。
   * 發布主題開發規格，指導使用者如何搭配 base 樣式表。
